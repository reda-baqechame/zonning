# Competitor UX And Workflow Scorecard

Date: 2026-06-21
Scope: first-10-minute Quebec contractor teardown

## Goal

Measure how fast a Quebec contractor can find, trust, qualify, save, and act on a relevant opportunity in each competitor.

## Products To Test

- SEAO.
- MERX/SOVRA.
- Ontopical/SOVRA.
- ConstructConnect.
- Biddingo.
- bids&tenders / municipal procurement portals.
- GovBid.
- BidFit/BidFilter-style tools.
- ProcureData.
- Local municipal portals.

## Test Persona

Use the same persona for every product:

- Company: Quebec specialty subcontractor.
- Trade: electrical, HVAC, excavation, or concrete.
- Regions: Montreal, Laval, Quebec City, Monteregie.
- Goal: find one opportunity worth reviewing this week.
- Decision needed: chase, ignore, or watch.

## First-10-Minute Protocol

Record screen and time each step.

1. Open product home page or logged-in trial account.
2. Search for one English query and one French query.
3. Filter to Quebec.
4. Filter by construction/trade if possible.
5. Open the first promising opportunity.
6. Find buyer/owner/source.
7. Find deadline/stage.
8. Find documents or source links.
9. Find addenda/plan-holder/bidder/history if available.
10. Save, alert, export, or share it.
11. Decide chase/no-chase and write why.

## Queries

Use these consistently:

- "Montreal electrical tender"
- "Laval excavation"
- "Quebec City HVAC"
- "appel d'offres construction Montreal"
- "travaux excavation Laval"
- "ventilation Quebec"

## Scorecard

Rate 1 to 5.

| Dimension | 1 | 3 | 5 |
|---|---|---|---|
| Time to first relevant Quebec opportunity | Cannot find one | Found in 5-10 minutes | Found in under 2 minutes |
| Search clarity | Confusing | Usable with effort | Obvious and precise |
| Filter quality | Generic/noisy | Some useful filters | Trade, region, stage, buyer, deadline, value |
| French/Quebec fit | Weak | Partial | Native-feeling Quebec workflow |
| Trust/source visibility | Source unclear | Source visible after digging | Source, freshness, and authority are obvious |
| Document workflow | Missing/confusing | Documents available with friction | Clear documents, addenda, order/download/submission |
| Saved searches/alerts | Missing | Basic | Useful and configurable |
| Bid/no-bid support | No support | Some context | Strong fit, risk, and next-action guidance |
| Team collaboration | Missing | Share/save only | Workspace, notes, assignments, exports |
| Mobile usability | Broken | Usable for lookup | Useful for monitoring and triage |

## ZONNING Comparison Questions

After each competitor teardown, answer:

- What did this product do better than ZONNING should attempt today?
- What did this product do that ZONNING should not copy?
- What was missing for a Quebec contractor?
- Was the product stronger for tender access, early intelligence, compliance, document workflow, or project management?
- Which ZONNING claim would be false after seeing this competitor?
- Which ZONNING wedge still survives?

## Evidence To Capture

For each product:

- Date tested.
- Account type or public access type.
- Query used.
- Screenshot/video link.
- First relevant opportunity URL.
- Time to first relevant opportunity.
- Final score by dimension.
- Three UX strengths.
- Three UX weaknesses.
- "Do not compete here" note.
- "ZONNING opening" note.

## Expected Benchmarks

### SEAO

Expected strength:

- Official source trust, Quebec public tender authority, documents and notices.

Expected ZONNING opportunity:

- Faster contractor qualification, source-context summary, compliance and site intelligence.

### MERX/SOVRA

Expected strength:

- Mature tender discovery, alerts, document workflow, plan-holder/history tools, eBid seriousness.

Expected ZONNING opportunity:

- Quebec-specific permit/RBQ/AMP/RENA/site intelligence around opportunities.

### Ontopical/SOVRA

Expected strength:

- Pre-RFP local-government signal intelligence and outreach workflow.

Expected ZONNING opportunity:

- Narrower Quebec construction execution with permits, trades, licenses, and site context.

### ConstructConnect

Expected strength:

- Construction bid board, project filters, team workflow.

Expected ZONNING opportunity:

- Quebec official/regulatory/local context, not broad construction bid-board replacement.

### ProcureData

Expected strength:

- Normalized procurement data, API, searchability, procurement document extraction.

Expected ZONNING opportunity:

- Contractor-facing decision workflow instead of data infrastructure only.
